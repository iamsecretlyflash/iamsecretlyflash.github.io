Setting up a webpage like this

Jan 19, 2025

Just Why?

The first thing that needs to be addressed is why I decided to make this website. Why did I spend the whole of 19th January 2025 figuring out how to host this goddamn webpage?

The reason is threefold:

  • I have lately realized I love to yap, have tons of thoughts, and would love a place to pen (type) them down (hence the blog).
  • I have always been somewhat obsessed with the idea of having a domain name for myself. I splurged a bit today and hence had to add content to the page with my name—what better than random thoughts?
  • I have to thank Aniruddha Deb for the idea of hosting a blog page using Hugo. I have always enjoyed reading his blogs and was inspired by him to create something similar.

You will see that the theme of this page is inspired by Aniruddha’s page because I found Hugo through his blog and used some style elements from his GitHub repository.

Why this guide?

I have been trying to set up a page with Hugo for a very, very long time now, and until today, I had only failed. I bought a domain for Aries-IITD yesterday and hosted the club webpage there. I thought of doing the same for myself, so I bought my own domain XD. (Yup, I bought the domain before setting up the webpage.) This basically meant I had to set this up, or I would have been very disappointed in myself.

This guide assumes you are working with macOS and have Homebrew installed (if not, what the hell are you doing?). It’s a step-by-step guide, so it should be easy.

1. Install Hugo

brew install hugo

2. Create a new site

Run this in your projects directory.

  hugo new site blogpage
  cd blogpage

On navigating to your site you will find some pre-defined files and folders.

3. Adding a Hugo Theme

Add Hugo Paper as a submodule:

    git init
    git submodule add https://github.com/nanxiaobei/hugo-paper.git themes/hugo-paper

Update hugo.toml/config.toml

  theme = "hugo-paper"

4. Adding sections to the webpage

Adding homepage content

    vim content/_index.md

Paste content:

  ---
  title: "Welcome"
  draft: false
  ---
  Hi, I'm Vaibhav Seth! Welcome to my blog and research website.

5. Adding a blog page

    mkdir -p content/blog
    vim content/blog/_index.md

Add a description to the blog page landing (_index.md)

  ---
  title: "Blog"
  draft: false
  ---
  Welcome to my blog! Explore my latest thoughts and experiences.

Add a blog post:

    hugo new blog/first-post.md

Add content:

  ---
  title: "Quack"
  date: 2025-01-19
  draft: false
  ---
  Quack.

Other pages can be similarly added.

6. Navigation Links

We will be editing hugo.toml/config.toml

  [[menu.main]]
  name = "Home"
  url = "/"
  weight = 1

  [[menu.main]]
  name = "Blog"
  url = "/blog/"
  weight = 2

  [[menu.main]]
  name = "Papers"
  url = "/papers/"
  weight = 3

7. Testing and Deployment

To test locally, go to the root of the hugo project and run the following:

    hugo server

This will host a local website.

To deploy, run the following command in the same level:

    hugo

This will build files and add them to the public page.

To host this page on github pages run the following commands inside public.

    cd public
    git init
    git remote add origin https://github.com/<your-username>/<your-username>.github.io.git
    git add .
    git commit -m "Deploy Hugo site"
    git push -f origin main

The site will be live at: https://<your-username>.github.io/

Every time you make changes, make sure you build that static site by running Hugo command and doing a git push
